# Документация проекта "SmartCalc v1.0"

Данный проект является частью учебной программы Школы 21. В его рамках была реализована расширенная версия калькулятора на языке программирования Си с использованием структурного подхода. Помимо базовых арифметических операций, как плюс-минус и умножить-поделить, калькулятор имеет возможность вычисления арифметических выражений с учетом приоритетов, а так же некоторыми математическими функциями (синус, косинус, логарифм и т.д.). Помимо вычисления выражений калькулятор так же поддерживает использование переменной x и построение графика соответствующей функции. Логика и графический интерфейс были разделены, все модули покрыты тестами. Для визуальной части проекта был использован фреймворк Qt.

# Задание
- Программа должна быть разработана на языке Си стандарта C11 с использованием компилятора gcc. Допустимо использование дополнительных библиотек и модулей QT
- Код программы должен находиться в папке src
- Сборка программы должна быть настроена с помощью Makefile со стандартным набором целей для GNU-программ: all, install, uninstall, clean, dvi, dist, test, gcov_report. Установка должна вестись в любой другой произвольный каталог
- Программа должна быть разработана в соответствии с принципами структурного программирования
- При написании кода необходимо придерживаться Google Style
- Должно быть обеспечено покрытие unit-тестами модулей, связанных с вычислением выражений, с помощью библиотеки Check
- Реализация с графическим пользовательским интерфейсом, на базе любой GUI-библиотеки с API для C89/C99/C11
      <br/>Для Linix: GTK+, CEF, Qt
      <br/>Для Mac: GTK+, Nuklear, raygui, microui, libagar, libui, IUP, LCUI, CEF, Qt
- На вход программы могут подаваться как целые числа, так и вещественные числа, записанные через точку. По желанию можно обрабатывать ввод чисел в экспоненциальной записи
- Вычисление должно производиться после полного ввода вычисляемого выражения и нажатия на символ `=`
- Вычисление произвольных скобочных арифметических выражений в инфиксной нотации
- Вычисление произвольных скобочных арифметических выражений в инфиксной нотации с подстановкой значения переменной _x_ в виде числа
- Построение графика функции, заданной с помощью выражения в инфиксной нотации с переменной _x_ (с координатными осями, отметкой используемого масштаба и сеткой с адаптивным шагом)
  - Не требуется предоставлять пользователю возможность менять масштаб
- Область определения и область значения функций ограничиваются по крайней мере числами от -1000000 до 1000000
  - Для построения графиков функции необходимо дополнительно указывать отображаемые область определения и область значения
- Проверяемая точность дробной части - минимум 7 знаков после запятой
- У пользователя должна быть возможность ввода до 255 символов
- Скобочные арифметические выражения в инфиксной нотации должны поддерживать следующие арифметические операции и математические функции:

# Backend
Файлы "notation.h" и "stack.h" содержат определения структур и прототипы функций, необходимых для работы с инфиксными выражениями и преобразования их в обратную польскую нотацию в проекте. Здесь определены структуры узлов для работы со стеками и прототипы функций, используемых для управления стеками для значений типа char, а имнно формирования строки и типа double, для нахождения значения выражений.

Файл включает в себя функции:
1. Очистки стека, добавления и удаления узлов, проверки на пустоту стека для типа данных double.
2. Проверки длины строки и валидности скобок, символа на цифру и строки в целом.
3. Определяения приоритета математических операций.
4. Преобразования инфиксной строки в обратную польскую нотацию.
5. Выполнения операций и функций математических выражений, включая арифметические операции, модуль, возведение в степень и тригонометрические функции.
6. Вычисления выражения в обратной польской нотации.

Все функции в файлах "notation.h" и "stack.h" хорошо структурированы и оптимизированы для работы с различными типами данных. Данный функции выполняют ключевые операции для корректного анализа и вычисления математических выражений. Это позволяет удобно использовать их в основной логике проекта для обработки пользовательского ввода и генерации результата вычислений в правильной форме.

Файл "tests_smart_calc.c" содержит в себе тесты для проверки реализации основной части проекта и выполнение математических операций. Эти тесты помогают убедиться в корректности работы функций стека и вычислений, а также валидации входных данных при разборе математических выражений. Таким образом, файл "tests_smart_calc.c" играет ключевую роль в качестве инструмента для обеспечения стабильной работы функционала программы и поддержания ее качества. 

# Frontend
![SmartCalc](/misc/rus/images/be57bca6-fa71-400b-b494-f2ae7369bded.jpeg)  
Данная часть проекта, написанная с использованием Qt на языке C++, она отвечает за визуальное представление проекта и
включает в себя создание интерфейса пользователя с помощью различных виджетов и элементов управления, таких как кнопки, поля ввода и отображения результатов. Frontend обрабатывает пользовательский ввод, передает данные в Backend для обработки, а затем отображает результаты обратно пользователю.

Файл "mainwindow.cpp" содержит определение класса MainWindow, который представляет собой основное окно приложения калькулятора. В этом файле определены методы, обработчики событий (слоты) и подключения сигналов к ним, которые отвечают за функциональность калькулятора. Данный файл реализует следующую функциональность:
1. Управление виджетами на основном окне калькулятора, такими как кнопки, текстовые поля, графики и метки.
2. Установка ограничений на ввод и вывод данных.
3. Обработка нажатий кнопок для формирования строки в инфиксной нотации.
4. Обновление значений минимумов и максимумов для графика функций.
5. Подсчет результата выражения в инфиксной нотации.
6. Отображение графика функции при наличии переменной x в выражении.

Таким образом, данный файл играет ключевую роль в реализации логики и пользовательского интерфейса калькулятора, помогая обеспечить его работоспособность и функциональность.

Файл "mainwindow.h" является заголовочным файлом класса MainWindow в проекте. В нем объявляются методы класса, а также подключаются необходимые библиотеки и внешние зависимости. Данный заголовочный файл содержит объявление класса MainWindow, который представляет главное окно приложения калькулятора. Он наследуется от QMainWindow и содержит различные методы и слоты для управления интерфейсом и логикой приложения. Файл содержит объявления методов, которые отвечают за обработку событий, взаимодействие с пользовательским интерфейсом и выполнение вычислений при работе с калькулятором. Эти методы включают в себя:
1. Метод "digits_numbers()" - для отслеживания нажатых кнопок и формирования строки в инфиксной нотации.
2. Метод "on_pushButton_cleaning_clicked()" - для очистки поля ввода на калькуляторе.
3. Метод "update_graph_range()" - для связывания значений минимумов и максимумов для графика функций.
4. Метод "calculate_result()" - для записи инфиксной строки в переменную и подсчета результата математического выражения.
5. Метод "display_graph_with_x()" - для отображения графика функции при наличии переменной x в выражении.

В файле происходит импорт необходимых библиотек Qt (QMainWindow, QString, QIntValidator) для работы с графическим интерфейсом, а также внешних заголовочных файлов, таких как stack.h и notation.h, которые, содержат реализацию стека и алгоритма преобразования выражения инфиксной нотации в ортаную польскую и подсчета результата выражения. Данный заголовочный файл играет важную роль в определении интерфейса и логики работы калькулятора, описывая структуру класса MainWindow и его функциональность.

Отрисовка графиков функций реализованная при помощи библиотеки "QCustomPlot". "QCustomPlot" - это библиотека визуализации для Qt, которая предоставляет возможность создания настраиваемых графиков простым способом. Она позволяет легко встраивать графику в приложения Qt и предоставляет разнообразные инструменты для настройки внешнего вида графиков, включая различные типы графиков, оси координат, легенду, стили линий и маркеров, сетку и многое другое.

# Установка
Для установки проекта подребуются следующие утилиты:
1. gcc
2. g++
3. make 
4. qmake

Makefile цели:  
1. "make test" - проверка части "Backend" при помощи Unit-тестов.
2. "make gcov_report" - формирование отчета в виде html старницы для отображения покрытия всех функций Unit-тестами.
3. "make install" - установка программы в папку "build".
4. "make dvi" - открытие документации по проекту.
5. "make dist" - генерация архива с исполняемым файлом и необходимой документацией в папке "dist".
6. "make clean" - отчиска скомпилированных файлов части "Backend".
7. "make uninstall" - отчистка установленных файлов проекта и созданного архива.
